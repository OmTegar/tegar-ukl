debian 11 Virtual Host

Posted by Tegar Dito

Om Tegar- Linux Debian 11 Virtual Host

sebelum masuk ke tutorial Virtual LAN Trunking debian 11, pastikan anda sudah mengikuti langkah langkah sebelum nya atau click disini jika di rasa kalian sudah mengikuti tutorial dengan baik. silahkan di coba untuk melakukan Virtual host.

Virtual Host adalah kelanjutan dari Virtual Lan Trunking jadi bagi temen temen yang belum mengikuti langkah langkah sebelumnya silahkan click link tersebut. Tetap di root terminal debian 11 , pertama tama pastikan CDROM debian sudah di aktifkan, lalu langkah berikutnya ubah network debianyang sudah kita ganti menjadi bridge adapter kita kembalikan kembali menjadi NAT. untuk tau caranya bisa click disini jika sidah silahkan restart networking.

systemctl restart networking

lalu kita install apache2 dengan mengetikkan "apt-get install apache2" dan tunggu hingga selesai.

apt-get install apache2

systemctl status apache2

selanjutnya kita matikan kembali CDROM dan kita kita kembalikan network menjadi bridge adapter dan jangan lupa kita restart networking kita

systemctl restart networking

jika apache2 sudah aktif click ctrl + c untuk keluar dan selanjutnya masukkan ip kalian di firefox, dengan cara masuk pada activities lalu click firefox atau bisa di cari di search bar.

Pastikan ip kita sudah terhubung ke apache2.

Kita akan Masuk pada folder html kita /var/www/html

cd /

cd /var/www/html

ls

normalnya ketika di "ls" file yang muncul hanyalah index.html , jadi kita akan buat folder kita terlebih dahulu dengan perintah sebagai berikut:

mkdir tegarT533 <--(nama folder)

cd tegarT533

nano index.html

kita mengetikkan html kita dengan di dalam file index html tersebut jika sudah , kalian bisa ctrl + x untuk simpan perubahan dan klik Y untuk menyetujui perubahan tersebut. kita akan kembali ke firefox dan silahkan ketikan ip address kalian , dan pastikan apa yang kalian tulis dalam folder html sudah tampil pada firefox.

192.168.1.33/tegarT533

jika terjadi error silahkan cek network kalian apakah sudah di bridge adapter atau belum , lalu masukkan perintah. "ip a" untuk cek ip kalian , dan jika di rasa sudah benar silahkan cek index html kalian.

Selanjutnya Kita akan konfigurasi apache2, hal ini dilakukan agar kita tidak perlu menuliskan nama folder setelah ip kita. dan hal ini dilakukan agar kita bisa menambahkan domain kita pada blog selanjutnya. okehh kita masuk ke folder apache2 terlebih dahulu.

cd /

cd /etc/apache2/sites-available

ls

Normalnya Folder tegarT533.conf masih belum ada , dan isi dari folder tersebut adalah file copy dari 000-default.conf jadi bagaimana cara untuk membuat foldernya?

cp 000-default.conf tegarT533.conf <-- nama file copy

nano tegarT533.conf

normalnya noname.com adalah localhost jadi nama ip kita, di baris kedua jangan lupa tambahkan nama folder kalian di /var/ww/html kalian

Server admin webmaster@noname.com <-- nama domain

DocumentRoot /var/www/html/tegarT533 <-- nama folder di var/www/html

serveralias www.noname.com <-- nama domain

Setelah kita konfigurasi di sites-available selesai silahkan klik "ctrl + x" lalu "Y" dan kita akan lanjut untuk copy folder kita di folder sites-enabled dengan cara

cp tegarT533.conf /etc/apache2/sites-enabled lalu kita cek folder sites sites-enabled

cd ../sites-enabled

ls

jika folder tegarT533.conf sudah muncul di sites enabled silahkan hapus file 000-default.conf

rm 000-default.conf

ls

systemctl restart apache2

jika dirasa sudah melakukan konfigurasi apache2 tanpa masalah silahkan kembali ke firefox dan coba masukkan pada search bar hanya ip saja seperti contoh di atas konfigurasi apache2 sudah selesai dan silahkan simak blog selanjutnya ya.
